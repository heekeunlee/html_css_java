웹 앱을 만드는 마법의 레시피

HTML, CSS, 그리고 JAVA(JavaScript)가 어떻게 조화를 이룰까요?

🏗️

HTML

"집의 뼈대와 벽"

웹사이트의 구조를 만듭니다. 제목, 본문, 이미지, 버튼 등 무엇이 어디에 있을지 결정하는 설계도와 같습니다.

  • 제목 (h1, h2)
  • 단락 (p)
  • 목록 (ul, li)
  • 이미지 (img)
🎨

CSS

"인테리어와 페인트"

웹사이트의 디자인을 담당합니다. 색상, 글꼴, 간격, 배치 등을 아름답게 꾸며주는 드레스업 과정입니다.

  • 배경 색상
  • 폰트 스타일
  • 레이아웃 (Flex, Grid)
  • 애니메이션
⚡

JAVA / JS

"전기 장치와 가전제품"

웹사이트의 동작과 로직을 담당합니다. 버튼을 눌렀을 때의 반응, 데이터 처리 등 살아있는 기능을 부여합니다.

  • 클릭 이벤트
  • 데이터 계산
  • 서버 통신
  • 복잡한 로직

체험해보기: 스위치를 켜보세요!

📚 웹 개발 용어 사전

모르는 용어가 나오더라도 당황하지 마세요! 비유로 쉽게 풀어서 설명해 드릴게요.

HTML (HyperText Markup Language)

비유: 건물의 뼈대와 벽돌

웹사이트의 가장 기초가 되는 구조를 잡는 언어입니다. 제목, 본문, 사진이 들어갈 자리를 정합니다.

예시: 신문 기사에서 제목(`h1`), 문단(`p`), 그리고 기사에 삽입된 사진(`img`)의 위치를 지정하는 것과 같습니다.

CSS (Cascading Style Sheets)

비유: 인테리어 소품과 화장

뼈대만 있는 웹사이트를 예쁘게 꾸며줍니다. 색상, 글꼴, 배경, 그리고 배치(레이아웃)를 담당합니다.

예시: "제목 글씨는 파란색으로 하고, 배경에는 은은한 그림자를 넣어줘!"라고 명령하는 디자인 가이드북입니다.

JavaScript (JS)

비유: 가전제품의 작동 버튼과 뇌

사용자와 상호작용하며 웹사이트가 살아 움직이게 만듭니다. 클릭하면 알림창이 뜨거나 정보를 계산하는 로직을 담습니다.

예시: 쇼핑몰에서 '장바구니 담기' 버튼을 눌렀을 때, 개수가 1 늘어나고 "담겼습니다!"라고 메시지가 뜨는 기능을 만듭니다.

React (리액트)

비유: 조립식 레고 세트 (라이브러리)

페이스북이 만든 현대 웹 개발의 대세 도구입니다. 화면을 조각조각(컴포넌트)으로 나누어 만들어 재사용합니다.

예시: '로그인 창'이라는 레고 블록을 한 번 만들어두면, 메인 페이지에서도 마이페이지에서도 그대로 가져다 꽂기만 하면 됩니다.

Node.js (노드제이에스)

비유: 브라우저 밖으로 나온 자바스크립트

원래 웹 브라우저 안에서만 놀던 JavaScript를 컴퓨터 시스템 어디서든(서버 등) 일할 수 있게 해주는 환경입니다.

예시: 카페(브라우저)에서만 서빙하던 직원이 이제 카페 밖 배달 업무(서버 처리)까지 할 수 있게 된 것입니다.

Database (DB / 데이터베이스)

비유: 거대한 보관 창고

회원 정보, 게시글 내용, 주문 내역 등을 안전하게 저장하고 나중에 다시 꺼내볼 수 있도록 관리하는 창고입니다.

예시: 로그인할 때 "내가 입력한 비밀번호가 창고(DB)에 저장된 비밀번호랑 똑같은가?"를 확인할 때 사용합니다.

API (Application Programming Interface)

비유: 식당의 '점원(웨이터)'

서로 다른 프로그램들이 정보를 주고받을 수 있게 연결해주는 창구입니다.

예시: 손님(프론트엔드)이 점원(API)에게 스테이크(데이터)를 주문하면, 점원이 주방(백엔드)에 가서 음식을 가져다주는 통로 역할을 합니다.

Git & GitHub (깃 & 깃허브)

비유: 게임 저장(Save) 포인트와 공유 도서관

Git은 코드의 수정 이력을 기록하는 타임머신이고, GitHub은 그 기록들을 온라인에 올려서 공유하는 장소입니다.

예시: "어제 코드로 돌아가고 싶어!" 할 때는 Git을 쓰고, "내 코드를 다른 사람에게 보여줄래!" 할 때는 GitHub을 씁니다.

Framework vs Library

비유: 다 지어진 아파트(프레임워크) vs 공구 세트(라이브러리)

프레임워크는 틀이 정해져 있어 그 안에서 일하는 것이고, 라이브러리는 필요한 도구만 골라 쓰는 것입니다.

예시: 밀키트(프레임워크)는 순서대로만 하면 요리가 완성되지만, 소금/설탕(라이브러리)은 내가 하고 싶은 요리에 자유롭게 넣습니다.
